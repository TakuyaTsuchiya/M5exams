# M5実習課題 設計仕様書

## 受講者プロフィール
- AIスクール受講生。ChatGPT, Midjourney, Canva, Higgsfield等のITツール経験あり
- **中級者**扱い。用語の基礎説明は最小限でよい
- M1〜M4でAI活用の基本は学習済み（議事録作成等はM1/M2で経験済み）

## トーン・文体
- カッチリ寄り。「〜しましょう」「〜してください」レベル
- 一文を短く、冗長な理屈・説教調は削る
- 「やる前に語る」より「さっさと作ろう」感を重視

### わかりやすさのルール
1. **一文目で「何をするか」を言い切る** ― 問いかけ（「〜していませんか？」）や学習意義（「〜の力を身につけましょう」）から入らない。「この課題では〜を作成します。」でOK
2. **読めばわかることを書かない** ― ラベルだけで伝わるなら括弧の補足は省く。テスト文を読めばわかる説明も不要
3. **説教しない・動機づけしない** ― 事実を書けば意義は読み手が汲み取る。「〜のスキルを身につけましょう」は不要
4. **一文一意** ― 一文に目的と手段と効果を全部詰めない。長い文は分割する
5. **見出しと文末を対応させる** ― 「目的」の下は「〜することです。」で結ぶ。見出しが名詞なら文末も名詞的に閉じる。さらに、その作業の先にある業務上のメリットまで書く

## 構成原則

### セクション間の積み上げ
- S1→S6は明確にステップアップしている。各課題文の冒頭で前セクションとの接続を意識させる
  - S1: Botを作る
  - S2: Botに知識を足す（RAG）
  - S3: Botに機能を足す（Web検索・画像生成）
  - S4: 事例を参考に自分の業務に合わせる
  - S5: 単発→ワークフローへ。業務を設計する
  - S6: 設計図を実際に動かす

### 迷わせない設計
- 「用途は自由です」ではなく、**具体的なお題を1つ指定する**
- テスト用のインプットテキストも**こちらで2〜3個用意する**
- 選択肢がある場合は選択肢を明示する（S4）

### GWS表記
- 「GWS」ではなく「Google Workspace」と表記する

## 評価方針（全課題共通）

### 最重要ルール
> **評価対象はプロンプト（カスタム指示・設計文）であり、AIの出力内容そのものは評価しない。**

これは採点者（マネージャー）向けのルブリックに記載する。受講生向けの課題文には記載しない。

### 評価の構造
| 評価する | 評価しない |
|---|---|
| カスタム指示の具体性 | 出力の文章の良し悪し |
| フォーマット指定の明確さ | キャッチコピーのセンス等 |
| 制約条件の設計 | AIの性能に依存する部分 |
| 出力フォーマットの安定性（プロンプト品質の間接指標として） | |

### 採点基準
- **30点満点**（全セクション共通）
- OK / 惜しい / NG の3段階
- 設計（プロンプト）に重い配点、結果（出力安定性）は間接指標としての配点

## 各セクション設計

### S1: Instagram投稿文を自動生成するSNS投稿Botを作ろう

**シラバスのゴール**: GPTs/Gemを作れるようになる。カスタム指示で出力を統一する

**課題でやること**: SNS投稿Botを1つ作り、こちらが用意した3つの入力テキストでテストし、フォーマットが安定することを確認する

**お題**: SNS投稿Bot（Instagram投稿文を生成）
- 議事録作成はM1/M2で経験済みのため、別のお題にした
- SNSのほうが「Botに作らせる面白さ」が伝わりやすい

**テスト用インプット（こちらで用意）**:
1. 商品情報（情報が揃っている）: 「夏の新作アイスティーが出ます。マンゴーとジャスミンのブレンド。税込450円。7月1日発売。冷たくてさっぱりした味わい。テイクアウト専用。」
2. イベント情報（内容が異なる）: 「来月、渋谷で無料ワークショップやります。AIを使ったデザイン入門。定員20名。講師は田中さん。初心者歓迎。」
3. 情報が少ない: 「新しいサービス始めます。詳細はまだ決まっていません。」

**Bot設定例（課題文に掲載）**:
- 役割: SNS運用担当アシスタント
- 出力フォーマット: キャッチコピー / 本文 / ハッシュタグ / 投稿タイミング案
- 制約条件: 体言止め、150文字以内、誇大表現禁止など

**採点基準（30点満点）**:
| 観点 | 配点 | OK | 惜しい | NG |
|---|---|---|---|---|
| 役割が明記されている | 6点 | 役割が具体的 | 書いてあるが曖昧 | なし |
| 出力フォーマットが具体的 | 6点 | 項目名・形式が明確 | 項目はあるが形式が曖昧 | 指定なし |
| 制約条件が設定されている | 6点 | 具体的なルールがある | 1つだけ or 曖昧 | なし |
| 3回の出力でフォーマット統一 | 12点 | 全回同じ構造 | 1回崩れた | バラバラ or 未テスト |

**提出物**:
1. Bot設定テキスト（Bot名・説明・カスタム指示の全文）
2. テスト入出力テキスト（指定された3つの入力に対する出力結果）
